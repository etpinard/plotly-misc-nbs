Plotly, matplotlib, and mpld3.

Quickstart

Matplotlylib lets you convert matplotlib figures to beautiful D3js graphs drawn online with Plotly's Python API. It's this easy...

>>> ## make an mpl figure ##
>>> import plotly.plotly as py
>>> py.plot_mpl(fig)

For install:

$ pip install plotly

Intro

The mplexporter package is a recent development as a part of the mpld3 project. The goal of mpld3 is to expand python's matplotlib use to the browser. By separating the matplotlib exporting and the actual graphic rendering, many renderers can now leverage the parsed matplotlib information. This makes the matplotlib-to-browser project easier for everyone. Nice.

Now, enter Plotly. Like keeping your data around? Want to share your plots and/or data online? We allow users to easily create and share interactive, publication-quality visualizations, right in a web browser. While Plotly's Python API gives users access to plotly directly, users may already have high-quality plots that they want to get online and share. Standing on the shoulders of the mplexporter, plotly is developing matplotlylib, a package that converts mpl figures to plotly figures with a simple call to plot_mpl or iplot_mpl (for plotting in IPython) . This function creates a JSON dictionary and sends it to be rendered on plotly with an option to put the resulting figure back into an IPython notebook.

If your data are important to you, they're important to us, and we'll help you share them with others.

For more infomation on using Plotly, you can check out our other IPython notebooks. Or, just take a look at some practical uses from Wired Science and The Washinton Post.

Start Plotting!

Simply make a figure and use the fig_to_plotly call.

In [41]: 
fig, ax = plt.subplots()

ax.set_title('hello world... wide web')
py.iplot_mpl(fig, strip_style=True, filename='hello mpl!')

Hover, zoom, pan--dang! That graph is live.

Strip style and resize

Let's get this out of the way early. While it's certainly true that you can make darn-fine looking graphs in matplotlib, it often takes a bit of effort.

Plotly's worked hard on its defaults, so why not give them a shot?

py.iplot(fig, strip_style=True)

Also, by default we resize mpl plots, but you can revert that:

py.iplot(fig, resize=False)

Line Plots

Alright, let's get going!

Here we instantiate a matplotlib figure object and fill it with stuff we want to plot! IPython's matplotlib magic function will catch this code block and pop it inline when you 'shift-enter'.

In [44]: 
fig1 = plt.figure()
x = np.arange(0,1.0,0.01)
y1 = np.sin(2*np.pi*x)
y2 = np.cos(4*np.pi*x)
ax = fig1.add_subplot(111)
ax.plot(x, y1, 'b--', label='sin')
ax.plot(x, y2, 'ro', label='cos')
ax.set_title("It's a Sign")
ax.set_xlabel('time')
ax.set_ylabel('amplitude')
Out[44]:
<matplotlib.text.Text at 0x10aa9bf50>

The following line takes the matplotlib figure object, extracts the data with mplexporter, renders a new plot with PlotlyRenderer, and sends it to Plotly. The plot shows up inline because of the notebook=True keyword argument. This causes the fig_to_plotly function to call plotly.iplot (IPython plot) instead of the normal plotly.plot function. Taking that keyword argument out defaults to the normal plot function and will open a new window in your browser with the plot.

Note how you can zoom, pan, autoscale, and see data points by hovering. The data you plotted is right there. If you want to modify the dataset, style, layout, or add notes, you can click the 'data and graph' link below the plot.

And again, if you have a plotly profile, you can save and share your awesome sinusoids with others.

In [45]: 
py.iplot_mpl(fig1, strip_style=True, filename="a sign")

Subplots

But why have one axes object when you could have five!? Gridspec is a handy way to intuitively partition figure space into nice looking subplots. Here are just some simple line plots stretching over a 3x3 grid of plotting areas.

In [46]: 
fig2 = plt.figure() # matplotlib.figure.Figure obj
gs = gridspec.GridSpec(3, 3)
ax1 = fig2.add_subplot(gs[0,:])
ax1.plot([1,2,3,4,5], [10,5,10,5,10], 'r-')
ax2 = fig2.add_subplot(gs[1,:-1])
ax2.plot([1,2,3,4], [1,4,9,16], 'k-')
ax3 = fig2.add_subplot(gs[1:, 2])
ax3.plot([1,2,3,4], [1,10,100,1000], 'b-')
ax4 = fig2.add_subplot(gs[2,0])
ax4.plot([1,2,3,4], [0,0,1,1], 'g-')
ax5 = fig2.add_subplot(gs[2,1])
ax5.plot([1,2,3,4], [1,0,0,1], 'c-')
gs.update(wspace=0.5, hspace=0.5)

Again, if you're fond of matplotlib's syntax, or, you're just feeling like dipping your toes in the plotly pool for now, inserting a simpe iplot_mpl call sends the above figure to Plotly and it responds by sticking back into this notebook.

By default, matplotlib subplots don't have common axis elements and so the rendered plotly graphic will reflect this. However, as we've mentioned, plotly's all about sharing. You can check out the possibilities for anchoring different subplots to the same axis in the plotly docs if you're trying to work through the API. Alternatively, you can use ploty's GUIs to change all of the plot's attributes--including axis sharing.

ax.text(0.999, 0.999, 'top-right', transform=ax.transAxes, va='top', ha='right')

Plotly will import these and anchor them appropriately to the plot, not to the data. You can pan and zoom in the following figure to check this. If you decide you don't want them anchored to the 'page', you can later switch them via the 'Notes' tab in Plotly.

Let's try a more interesting example. The following code creates a figure with a bunch of different line objects, each annotated with a text object that is anchored to data. Another annotation is anchored to the mpl axes objects so that it will remain in the top-left corner of the plot when you pan or zoom.

In [50]: 
gs = gridspec.GridSpec(2, 1)
fig4 = plt.figure()
ax1 = fig4.add_subplot(gs[0,0])
x = np.arange(-3, 3, 0.01)*2*np.pi
sin_x = np.sin(x)
ax1.plot(x, sin_x, 'k-', label='sin(x)')
num_lines = 20
colors = cm.rainbow(np.linspace(0, 1, num_lines))
for iii in range(0, num_lines):
    y = x
    for jjj in range(1, iii+1):
        sign = (-1)**jjj
        deg = 1+2*jjj
        y = np.add(y, sign*x**(deg)/np.math.factorial(deg))
    ct_h = int(len(x)/2)
    while abs(y[ct_h]) < 10:
        ct_h += 1
    ct_l = int(len(x)/2)
    while abs(y[ct_l]) < 10:
        ct_l -= 1
    x_plot = x[ct_l:ct_h+1]
    y_plot = y[ct_l:ct_h+1]
    ax1.plot(x_plot, y_plot, color=colors[iii], label='L{}'.format(iii))
    ax1.text(x_plot[-15], y_plot[-15], '{}'.format(1+2*iii), ha='center', va='center')
ax1.set_ylim(-10,10)
ax1.set_xlabel('x')
ax1.set_ylabel('y(x)')
ax1.set_title('approximation to sin(x)', size=14)
ax1.text(0.001, 0.999, 'num=deg', transform=ax1.transAxes, ha='left', va='top')

ax2 = fig4.add_subplot(gs[1,0])
x = np.arange(-5, 5, 0.01)
ax2.plot(np.arange(0.01, 5, 0.01),
         np.log(np.arange(0.01, 5, 0.01) + np.ones(len(np.arange(0.01, 5, 0.01)))),
         'k-', label='ln(x + 1)')
num_lines = 20
colors = cm.rainbow(np.linspace(0, 1, num_lines))
for iii in range(0, num_lines):
    y = x
    for jjj in range(1, iii+1):
        sign = (-1)**jjj
        deg = 1+jjj
        y = np.add(y, sign*x**(deg)/deg)
    ct_h = int(len(x)/2)
    while abs(y[ct_h]) < 3:
        ct_h += 1
    ct_l = int(len(x)/2)
    while abs(y[ct_l]) < 3:
        ct_l -= 1
    x_plot = x[ct_l:ct_h+1]
    y_plot = y[ct_l:ct_h+1]
    ax2.plot(x_plot, y_plot, color=colors[iii], label='L{}'.format(iii))
    ax2.text(x_plot[-28], y_plot[-28], '{}'.format(1+iii), ha='center', va='center')
ax2.set_ylim(-3,3)
ax2.set_xlim(-2,4)
ax2.set_xlabel('x')
ax2.set_ylabel('y(x)')
ax2.set_title('approximation to ln(1 + x)', size=14)
ax2.text(0.001, 0.999, 'num=deg', transform=ax2.transAxes, ha='left', va='top')
gs.update(wspace=0.2, hspace=0.5)

Bars

We like bar charts and histograms just as much as you do. Therefore, we've grabbed those for you too! Bars from bar charts are stored as 'patches' which are defined by paths. They are not, however, stored as a 'path_collection' since each bar is differently sized (or it should be unless you're making pretty boring bar charts). Plotly uses some educated guesses to decide whether rectangle patches that end up in each axes object are intended to be bar charts.

In [52]: 
fig7 = plt.figure()
ax = fig7.add_subplot(111)
left = [1, 2, 3, 4, 5, 6, 7]
height = [5, 6, 1, 4, 3, 10, 15]
width = 1
ax.bar(left, height, width=width)

Scatter

Scatter plots are a pretty common usage for matplotlib. Information about scatter plots is stored in something called a 'path_collection' which we needn't get into. The actual markers for scatter plots are stored as Bezier curves. The PlotlyRenderer will attempt to figure out what marker you've entered and import that into plotly. 'Nuff said.

Here's an example.

In [56]: 
fig9 = plt.figure()
ax = fig9.add_subplot(111)
num = 1000
s = 121
x1 = np.linspace(-0.5,1,num) + (0.5 - np.random.rand(num))
y1 = np.linspace(-5,5,num) + (0.5 - np.random.rand(num))
x2 = np.linspace(-0.5,1,num) + (0.5 - np.random.rand(num))
y2 = np.linspace(5,-5,num) + (0.5 - np.random.rand(num))
x3 = np.linspace(-0.5,1,num) + (0.5 - np.random.rand(num))
y3 = (0.5 - np.random.rand(num))
ax.scatter(x1, y1)
ax.scatter(x2, y2)
ax.scatter(x3, y3)

The Nitty Gritty

Out of curiosity or for the sake of troubleshooting, you might find yourself needing to inspect the dictionaries and possibly even the mplexporter. The following shows how you go about running the PlotlyRenderer and making a manual call to plotly.

In [58]: 
fig12 = plt.figure()
ax = fig12.add_subplot(111)
ax.plot([10,20,30],[100,200,300], 'bo')
ax.set_xlim(5, 35)
ax.set_ylim(90, 310)
Out[58]:
(90, 310)

First, we will need access to the Exporter and PlotlyRenderer classes, so we import them! The Exporter crawls through the matplotlib figure, judiciously grabs pertinent information, and invokes the renderer with which it was instantiated. The PlotlyRenderer accepts calls from the Exporter and creates an appropriate JSON dictionary for use with plotly.

In [59]: 
from plotly.matplotlylib import Exporter, PlotlyRenderer
renderer = PlotlyRenderer()
exporter = Exporter(renderer)
exporter.run(fig12)

We can look at the resulting list of data and layout dictionaries by looking at the renderer's data and layout attributes.

Finally, to create the figure in plotly, we can call plotly.iplot. Again, we'll need a couple more tools from plotly, so we'll import them. We instantiate our plotly class and call it py by entereing our username and api_key. Then we use plotly's iplot method to drop our plot back into this IPython notebook. For more information about plotly and IPython, Plotly's IPython docs have you covered.

In [61]: 
py.iplot(renderer.plotly_fig, filename='nitty-gritty', fileopt='overwrite')

Additionally, you can use this opportunity to add to or modify the data and layout dictionaries. Just as an example, we can change the plot and paper background colors. You might find this procedure helpful if you've snooped around the JSON drop-down in plotly on some of your favorite plots from the plotly gallery. If you see a feature you want, just add it!

In [62]: 
renderer.resize()
renderer.plotly_fig['layout']['plot_bgcolor'] = '#80adb0'
renderer.plotly_fig['layout']['paper_bgcolor'] = '#fefafd'
renderer.plotly_fig['layout']['showgridlines'] = False
py.iplot(renderer.plotly_fig)
